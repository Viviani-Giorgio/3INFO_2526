Localhost

In computer networking, localhost è un hostname che fa riferimento al computer attualmente in uso per accedervi. Viene utilizzato per accedere ai servizi di rete che sono in esecuzione sull'host tramite l'interfaccia di rete loopback. L'utilizzo dell'interfaccia loopback bypassa qualsiasi hardware di interfaccia di rete locale.

Loopback

Il meccanismo di loopback locale può essere utilizzato per eseguire un servizio di rete su un host senza richiedere un'interfaccia di rete fisica, o senza rendere il servizio accessibile dalle reti a cui il computer potrebbe essere connesso. Ad esempio, un sito web installato localmente può essere accessibile da un browser web tramite l'URL http://localhost per visualizzare la sua home page.

Il nome localhost normalmente risolve all'indirizzo loopback IPv4 127.0.0.1 e all'indirizzo loopback IPv6 ::1.

Name Resolution

Gli standard di rete IPv4 riservano l'intero blocco di indirizzi 127.0.0.0/8 (più di 16 milioni di indirizzi) per scopi di loopback. Ciò significa che qualsiasi pacchetto inviato a uno di quegli indirizzi viene reindirizzato. L'indirizzo 127.0.0.1 è l'indirizzo standard per il traffico loopback IPv4; gli altri non sono supportati da tutti i sistemi operativi. Tuttavia, possono essere utilizzati per configurare più applicazioni server sull'host, tutte in ascolto sulla stessa porta.

La risoluzione del nome localhost in uno o più indirizzi IP è normalmente configurata dalle seguenti righe nel file hosts del sistema operativo: 

127.0.0.1    localhost
::1          localhost

IETF Standards

Il nome localhost è riservato per scopi di loopback dallo standard RFC 6761 (Special-Use Domain Names), che ha raggiunto il livello di maturità Proposed Standard nel febbraio 2013. Lo standard stabilisce una serie di considerazioni speciali che governano l'uso del nome nel Domain Name System.

Packet Processing

L'elaborazione di qualsiasi pacchetto inviato a un indirizzo loopback è implementata nel livello di collegamento dello stack TCP/IP. Tali pacchetti non vengono mai passati a nessun controller di interfaccia di rete (NIC) o driver hardware e non devono apparire all'esterno di un sistema informatico, né essere instradati da alcun router.

Special Cases

Le versioni del database MySQL differenziano l'uso dell'hostname localhost dall'uso degli indirizzi 127.0.0.1 e ::1. Quando si utilizza localhost come destinazione in un'interfaccia di connessione client di un'applicazione, l'API di programmazione MySQL si connette al database utilizzando un Unix domain socket, mentre una connessione TCP tramite l'interfaccia loopback richiede l'uso diretto dell'indirizzo esplicito.
